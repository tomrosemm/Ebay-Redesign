How do I Use the Calculator?

First, start by adding rows until the desired number of total tasks in your task list is achieved Then, give each of them appropriate names, the successor/predecessor lists should populate themselves as these tasks are named. Then, use the selection menus to give predecessors, the task(s) that come immediately before the task in the ordering, and successors, the tasks that come after. To enter multiple selections, hold the ctrl key (Windows) or cmd key (MacOS) while selecting. After selections are made, the selections will not be visible, there's no cause for concern, they should still be saved. Both predecessors and successors must be selected. Leaving one or the other empty will result in invalid data for boundary time calculations. Afterwards, enter durations of each task in a consistent unit of time (hours, days, weeks) and press calculate. The right side of the table should fill up with boundary time information, explained in the section "What Do These Terms Mean?", and the bottom of the screen should display an activity graph of tasks, with a critical path highlighted in red. Additionally, there's a toggle at the bottom-left to display additional information on the graph.

What Do These Terms Mean?

EST: Earliest Start Time, The earliest time unit a task can start at, assuming all other tasks finish on time. EFT: Earliest Finish Time, the earliest a task is able to finish. LST: Latest Start Time, the latest time unit that a task is able to finish in order for the project to avoid unecessary halts. LFT: Latest Start Time, the latest a task can possibly finish.

What IS a Critical Path?

A critical path represents all of the tasks in an ordering with slack time of 0. This means that the time of the entire task list is determined by the time it takese to complete these tasks, and thusly they are the most vital tasks to prioritize as the project's schedule revolves around them.

Why Is a Critical Path Useful to Me?

Throughout a wide variety of project management tasks, a critical path will ensure that a project stays on schedule by prioritizing those tasks that are unmovable for the overall structure of the project.
